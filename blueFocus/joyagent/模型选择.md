根据代码分析，JoyAgent-JDGenie的ReAct Agent使用了多种大语言模型，具体配置如下：

## 主要模型配置

### ReAct Agent模型
ReAct Agent默认使用 `claude-3-7-sonnet-v1` 模型 [1](#0-0) 。在 `ReactImplAgent` 的构造函数中，通过配置获取模型名称： [2](#0-1) 

### 其他Agent模型
- **PlanningAgent**: 默认使用 `gpt-4.1` [3](#0-2) 
- **ExecutorAgent**: 默认使用 `gpt-4.1` [4](#0-3) 

## 模型配置系统

### 配置文件结构
系统通过 `GenieConfig` 类管理模型配置，支持为不同Agent类型配置不同模型： [5](#0-4) 

### LLM设置映射
系统还支持详细的LLM配置，包括API密钥、基础URL、最大token数等： [6](#0-5) 

例如，`claude-3-7-sonnet-v1` 的详细配置： [7](#0-6) 

## 工具组件中的模型

### Code Interpreter
代码解释器工具默认使用 `claude-3-5-sonnet` 模型： [8](#0-7) ，在代码中通过环境变量获取： [9](#0-8) 

### 搜索组件
搜索工具的各个组件也使用 `gpt-4.1` 模型：
- 查询分解： [10](#0-9) 
- 搜索推理： [11](#0-10) 
- 答案生成： [12](#0-11) 

## Notes

系统采用了灵活的模型配置架构，允许为不同的Agent和工具组件配置不同的模型。ReAct Agent作为核心组件使用了Claude 3.7 Sonnet模型，而其他组件主要使用GPT-4.1模型。所有模型配置都支持通过配置文件或环境变量进行自定义。
