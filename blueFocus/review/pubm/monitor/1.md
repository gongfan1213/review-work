## 🎯 **面试话术：监控模块（pmhub-monitor）**

### **1. 简单来说，我做了什么？**
> "我负责搭建了一个**监控中心**，就像给整个系统装了一个'健康体检仪'，能实时看到所有微服务的运行状态，比如哪个服务挂了、哪个服务慢了，出了问题能第一时间发现。"

### **2. 用大白话解释技术原理**

#### **核心依赖（pom.xml）**
```xml
<!-- 这是监控的核心工具 -->
<dependency>
    <groupId>de.codecentric</groupId>
    <artifactId>spring-boot-admin-starter-server</artifactId>
</dependency>
```
**面试官问：为什么选Spring Boot Admin？**
- 答：因为它是一个**开箱即用**的监控工具，就像买了个现成的监控面板，不用自己写代码去收集各种指标。

#### **启动类（PmHubMonitorApplication.java）**
```java
@EnableAdminServer  // 开启监控服务
@SpringBootApplication
public class PmHubMonitorApplication {
    public static void main(String[] args) {
        SpringApplication.run(PmHubMonitorApplication.class, args);
    }
}
```
**面试官问：@EnableAdminServer是干什么的？**
- 答：这个注解就像按下了监控系统的"开关"，告诉Spring Boot："我要启动监控功能了"。

### **3. 监控系统是怎么工作的？**

#### **第一步：服务注册**
- 每个微服务（比如项目服务、工作流服务）启动时，会向Nacos注册中心"报到"
- 监控中心从Nacos那里知道有哪些服务在运行

#### **第二步：健康检查**
- 监控中心定期"打电话"给每个服务，问："你还好吗？"
- 如果服务回答"我很好"，状态就是UP（上线）
- 如果服务不回答或说"我病了"，状态就是DOWN（下线）

#### **第三步：状态通知**
```java
// 当服务状态发生变化时，会触发这个通知
public class PmHubStatusChangeNotifier extends AbstractStatusChangeNotifier {
    @Override
    protected Mono<Void> doNotify(InstanceEvent event, Instance instance) {
        return Mono.fromRunnable(() -> {
            if (event instanceof InstanceStatusChangedEvent) {
                String status = ((InstanceStatusChangedEvent) event).getStatusInfo().getStatus();
                switch (status) {
                    case "DOWN":  // 服务挂了
                        System.out.println("发送 健康检查没通过 的通知！");
                        break;
                    case "OFFLINE":  // 服务离线
                        System.out.println("发送 服务离线 的通知！");
                        break;
                    case "UP":  // 服务上线
                        System.out.println("发送 服务上线 的通知！");
                        break;
                }
            }
        });
    }
}
```

**面试官问：这个通知器是干什么的？**
- 答：就像个"报警器"，当发现服务状态异常时，会立即通知运维人员，比如发邮件、发短信，让问题能快速被发现。

### **4. 安全配置（WebSecurityConfigurer.java）**

```java
@EnableWebSecurity
public class WebSecurityConfigurer {
    @Bean
    public SecurityFilterChain filterChain(HttpSecurity httpSecurity) throws Exception {
        return httpSecurity
            .authorizeRequests()
            .antMatchers(adminContextPath + "/assets/**", 
                        adminContextPath + "/login",
                        adminContextPath + "/actuator/**",
                        adminContextPath + "/instances/**"
            ).permitAll()  // 这些路径允许所有人访问
            .anyRequest().authenticated()  // 其他路径需要登录
            .and()
            .formLogin()  // 启用登录表单
            .and()
            .csrf().disable()  // 关闭CSRF保护
            .build();
    }
}
```

**面试官问：为什么要做安全配置？**
- 答：监控中心能看到系统的"内部情况"，如果谁都能访问，那系统就暴露了，所以需要登录验证，只有授权人员才能看。

### **5. 配置文件（bootstrap.yml）**

```yaml
server:
  port: 6888  # 监控中心运行在6888端口

spring:
  application:
    name: pmhub-monitor  # 服务名称
  cloud:
    nacos:
      discovery:
        server-addr: 127.0.0.1:8848  # 连接Nacos注册中心
```

**面试官问：为什么用bootstrap.yml而不是application.yml？**
- 答：bootstrap.yml比application.yml先加载，这样监控中心就能先连接到Nacos，然后从Nacos那里获取其他配置。

### **6. 面试时怎么介绍这个模块？**

#### **开场白（30秒）**
> "我负责搭建了系统的监控中心，基于Spring Boot Admin实现，能实时监控所有微服务的健康状态、性能指标，并提供异常告警功能。"

#### **技术要点**
1. **Spring Boot Admin**：开箱即用的监控面板
2. **Nacos集成**：自动发现和注册微服务
3. **状态通知**：服务异常时自动告警
4. **安全控制**：登录验证，保护监控数据

#### **解决的问题**
- 服务挂了不知道
- 性能问题发现不及时
- 运维人员被动响应

#### **实现效果**
- 实时监控所有服务状态
- 异常情况主动告警
- 提供可视化的监控界面

### **7. 面试官可能追问的问题**

**Q：监控中心挂了怎么办？**
- A：监控中心本身也要做高可用，可以部署多个实例，或者用其他监控工具（如Prometheus）作为备选方案。

**Q：如何避免监控系统影响业务性能？**
- A：监控是异步进行的，不会阻塞业务请求；健康检查的频率可以调整，避免过于频繁。

**Q：监控数据怎么存储？**
- A：Spring Boot Admin默认是内存存储，生产环境可以配置持久化存储，比如数据库。

- **功能实用**：能解决生产环境的核心问题
- **扩展性好**：可以集成邮件、短信等通知方式
- **运维友好**：提供Web界面，非技术人员也能看懂
