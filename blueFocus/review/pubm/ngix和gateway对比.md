1、什么是网关，网关在微服务架构中的作用是什么？
- 微服务网关是一个位于微服务架构前端的组件，它充当了所有微服务的入口。
- 微服务网关负责路由请求、负载均衡、安全认证、流量控制、监控和日志记录等任务。
- 微服务网关可以将多个微服务组合成一个统一的API，从而简化了客户端与微服务之间的通信。

2、有Nginx了为什么还要SpringCloud Gateway做网关，两者有啥区别？
- Nginx在微服务架构中主要负责传输层的负载均衡，属于前置网关。
- SpringCloud Gateway从逻辑上更贴近服务侧，一般作为API网关，与项目更是一个整体。
- 可以根据业务更可塑地进行网关逻辑的编写和调度，主要作为应用层负载均衡（七层LB）。
- 但是SpringCloud Gateway是Java编写的，性能远不及Nginx。
- 当流量还未被接收时一次性的涌入SpringCloud会直接被冲垮。
- Nginx作为老牌的服务器网关，底层是C++。
- 性能上是绝对碾压SpringCloudGateway的存在，所以还需要前置的Nginx。
- 功能上，除了网关这个能力外，Nginx在很多情况下还充当静态资源服务器。
- Nginx不再承担着服务的路由分发和负载均衡，转而使用SpringCloudGateway做更专业的事。
- 是解耦的一种表现，两种网关的出现是架构在进化中对服务进一步细分的体现。

OSI七层模型、TCP/IP五层模型、TCP/IP四层模型对应关系如下：
| OSI七层模型 | TCP/IP五层模型 | TCP/IP四层模型 |
| ---- | ---- | ---- |
| 应用层 | 应用层 | 应用层 |
| 表示层 | - | - |
| 会话层 | - | - |
| 传输层 | 传输层 | 传输层 |
| 网络层 | 网络层 | 网络层 |
| 数据链路层 | 数据链路层 | 网络接口层 |
| 物理层 | 物理层 | - |

3、你是如何编写SpringCloud Gateway全局过滤器的？
- 在PmHub项目中，我单独在网关服务中新建了过滤器类AuthFilter。
- 实现了gateway的GlobalFilter接口，并自定义了filter方法实现。
- 方法里面主要是做了几个事情，分别是白名单过滤，过滤掉不需要认证的路径。
- 然后进行Token鉴权的校验，最后设置用户信息到请求，方便服务调用传递。
- 在接口内部还记录了访问接口的开始时间，方便统计接口调用耗时情况。
- AuthFilter还实现了spring的Ordered接口，并设置-200返回。
- 以用来提高AuthFilter在spring中的加载顺序为最先。

4、在自定义网关鉴权过程中，你遇到了哪些挑战？你是如何解决的？
1. 性能挑战：
- 在高并发场景下，网关的鉴权逻辑可能成为系统的瓶颈，导致请求延迟增加。
- 解决办法：使用缓存机制减少对数据库或认证服务器的调用次数，将用户的认证信息缓存一段时间。
2. 可扩展性：
- 随着系统的扩展，新的服务和认证方式可能会不断增加，如何保证鉴权系统的可扩展性是一个挑战。
- 解决办法：将鉴权逻辑独立为一个服务，方便独立扩展和维护。
3. 日志和监控：
- 在鉴权过程中，需要对用户的认证行为进行详细的日志记录和监控，以便在出现问题时进行排查。
- 解决方案：在关键的鉴权步骤添加详细的日志记录，记录请求、响应、错误等信息。
- 部署监控系统对鉴权服务的性能、错误率等进行实时监控。
