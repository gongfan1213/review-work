## 1. 总体结构

`SocialMediaSearchClient_XHS` 是一个专门用于小红书内容搜索的API客户端，支持“关键词搜索小红书帖子+评论”，并对结果进行结构化和简化处理。  
其核心流程包括：认证、任务创建、任务轮询、数据获取、数据过滤与格式化。

---

## 2. 主要方法与流程

### 2.1 初始化与认证

- 初始化时设置基础API地址（`self.base_url`），并创建认证客户端 `AuthClient`。
- 认证信息用于后续所有API请求的header。

### 2.2 创建搜索任务

- 方法：`create_search_task(keywords: str) -> str`
- 作用：向后端API发起“关键词搜索”任务，指定平台（media_id=4为小红书）、时间范围、数据类型、排序方式等。
- 返回：任务ID（`task_id`），后续用于查询任务状态和获取数据。

### 2.3 轮询任务状态

- 方法：`wait_for_task_completion(task_id: str, timeout: int = 300) -> bool`
- 作用：定时（0.5秒一次）查询任务状态，直到任务完成（状态码2），或失败/超时。
- 任务状态码：0=未运行，1=运行中，2=成功，3=失败，4=已取消。

### 2.4 获取任务数据

- 方法：`fetch_task_data(task_id: str, limit: Optional[int] = None) -> Dict`
- 作用：任务完成后，通过API获取搜索到的原始帖子数据（包含帖子、作者、评论等）。

### 2.5 数据处理与过滤

- 方法：`_process_search_result(raw_data: Dict) -> Dict`
- 作用：
  - 只保留有内容（content不为空）的帖子，最多5篇。
  - 对每篇帖子，仅保留指定字段（如标题、内容、封面、发布时间、作者等）。
  - 作者信息、评论信息也做字段过滤和结构化处理（如只保留前5条主评论及其子评论）。
- 结果结构：`{"docs": [ ... ]}`，每个doc包含`post_info`、`author_info`等。

### 2.6 简化格式输出

- 方法：`search_posts_simple_format(keywords: str) -> Dict`
- 作用：在完整搜索流程基础上，进一步简化输出，只保留最核心的字段（如标题、内容、作者、时间、url等），并统一为 `posts` 列表，便于上层业务直接使用。

### 2.7 完整搜索流程

- 方法：`search_posts_with_comments(keywords: str) -> Dict`
- 作用：串联上述所有步骤，完成一次“关键词搜索小红书内容”的全流程。
  1. 获取token
  2. 创建任务
  3. 轮询任务状态
  4. 获取原始数据
  5. 处理并返回结构化结果

## 3. 典型调用链

1. **初始化客户端**
   ```python
   client = SocialMediaSearchClient_XHS()
   ```
2. **发起搜索**
   ```python
   result = client.search_posts_simple_format("美食探店")
   ```
3. **返回结果**
   - 结构化的帖子列表，每条包含标题、内容、作者、时间、url等，便于直接用于AI生成、前端展示等。

## 4. 关键细节

- **认证安全**：每次请求都带有token，自动处理header。
- **任务异步**：采用“任务制”API，先创建任务再轮询，适合大数据量和异步处理。
- **字段过滤**：只保留业务需要的字段，减少冗余，提升效率。
- **异常处理**：每一步都有详细的日志和异常抛出，便于排查问题。
- **可扩展性**：如需更多字段或更复杂的处理，只需扩展过滤方法即可。
