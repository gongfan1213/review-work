### 四、Agents/Graph 分层与工作流（系统的“发动机”）
- 分层定义（按从底到上依赖）：
  - State 层：定义全局状态类型，工作流阶段常量，例如 `ContentWritingState`、`WorkflowStage`
  - Tool 层：原子能力（搜索、生成、审阅、并发分析、人工确认等）
  - Agent 层：组合工具与提示词策略（ReAct/Plan-Act），实现特定任务（需求对齐/选题研究/写作）
  - Node 层：把 Agent 包装为图节点，负责入参出参与状态读写
  - Workflow 层：以 `StateGraph` 编排节点与条件路由，定义起止与分支
  - Memory/Checkpoint 层：Memory/Redis 持久化，支撑容错/恢复/多轮
- 现成工作流示例：内容创作四阶段（Supervisor → 需求对齐 → 研究搜索 → 写作 → 人工确认路由）
- 搜索/写作 Agent 的提示与工具结合点非常明确，利于扩展更多平台（XHS、抖音、公众号等）
