### 📊 **主模型选择：Gemini 2.5 Pro**

#### **为什么选择 Gemini 作为主模型？**

```bash
# 环境配置
LLM_DEFAULT_PROVIDER=gemini
GEMINI_MODEL=gemini-2.5-pro
GEMINI_TEMPERATURE=0.5
GEMINI_MAX_TOKENS=16384
GEMINI_THINKING_BUDGET=512
```

**选择原因：**
1. **🧠 强大的推理能力**：支持 `thinking_budget` 参数，适合复杂思考任务
2. **🖼️ 多模态支持**：原生支持图片、文档等多模态输入
3. **💰 成本效益**：相比 GPT-4 更经济
4. **🏢 企业级稳定性**：Google 企业级服务，稳定性好
5. **📄 长上下文**：支持大 token 数，适合长文档处理

### 🧠 **Thinking Budget 策略（Gemini 独有）**

```python
def get_thinking_budget(self) -> int:
    """
    thinking_budget 参数仅在 Gemini 中有效
    OpenAI/Claude/通义千问等不支持
    """
    
    # 创作类和 hitpoint agent 使用高 thinking_budget (500)
    high_budget_agents = {
        'loomi_hitpoint_agent',
        'loomi_tiktok_script_agent', 
        'loomi_wechat_article_agent',
        'loomi_xhs_post_agent',
        'loomi_orchestrator'
    }
    
    if self.agent_name in high_budget_agents:
        thinking_budget = 500  # 高思考预算
    else:
        thinking_budget = 128  # 标准配置
```

**为什么这样配置？**
- 🧠 **复杂推理**：创作和分析任务需要更多思考时间
- ⚡ **性能平衡**：简单任务使用较少思考预算，提升响应速度
- 💰 **成本控制**：thinking_budget 影响 token 消耗，需要合理分配

### 🎯 **选择策略总结**

#### **模型选择原则**

1. **任务匹配原则**
   - 创作任务 → Gemini 2.5 Pro + 高 temperature + 高 thinking_budget
   - 分析任务 → Gemini 2.5 Pro + 低 temperature + 标准 thinking_budget
   - 简单任务 → Qwen Turbo + 低 temperature

2. **性能平衡原则**
   - 开发环境 → 快速模型优先
   - 生产环境 → 质量模型优先
   - 并发控制 → 环境差异化配置

3. **成本优化原则**
   - 复杂任务 → 高质量模型
   - 简单任务 → 低成本模型
   - 批量任务 → 经济型模型

4. **稳定性原则**
   - 主模型 → Gemini（企业级稳定性）
   - 备用模型 → 多提供商支持
   - 降级策略 → 自动切换机制

### 🏆 **技术优势**

1. **智能适配**：根据 Agent 类型自动调整参数
2. **环境感知**：开发/生产环境差异化配置
3. **成本控制**：任务复杂度与模型成本匹配
4. **性能优化**：并发控制 + 连接池预热
5. **可观测性**：集成 LangSmith/Langfuse 追踪

这种多模型策略设计使得 Loomi 系统能够在保证任务质量的前提下，最大化性能和成本效益，是一个典型的工程化 AI 系统模型选择方案。
