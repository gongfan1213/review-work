# DeepResearch 项目上下文管理详解

## 一、上下文的定义与作用

**上下文（Context）**，在 DeepResearch 中，指的是贯穿整个推理流程、动态累积和更新的所有“状态信息”，包括：
- 用户原始问题与历史消息
- 已采取的 action 及其结果
- 当前知识库（allKnowledge）
- 已发现的子问题/知识缺口（gaps）
- 评估反馈与反思内容
- URL 及外部检索内容
- Token 消耗与预算
- 允许的 action 状态
- 推理过程日志（diaryContext）
- 评估指标（evaluationMetrics）
- 其他如图片对象、引用、失败计数等

这些信息共同决定了每一轮推理的输入、决策和输出。

---

## 二、核心上下文数据结构

### 1. TrackerContext

```ts
const context: TrackerContext = {
    tokenTracker: existingContext?.tokenTracker || new TokenTracker(tokenBudget),
    actionTracker: existingContext?.actionTracker || new ActionTracker()
};
```
- **tokenTracker**：追踪token消耗，防止超预算。
- **actionTracker**：记录每步action、思考、gaps，便于复盘与调优。

### 2. 其他关键上下文变量

- **allContext**：`StepAction[]`，记录每一步的 action 及其详细信息。
- **allKnowledge**：`KnowledgeItem[]`，累积所有已获得的知识、反思、事实等。
- **gaps**：`string[]`，当前待解决的所有子问题/知识缺口。
- **diaryContext**：`string[]`，推理过程日志，便于反思与自我改进。
- **allQuestions**：`string[]`，所有已提出的问题（主问题+子问题）。
- **allKeywords**：`string[]`，所有已检索过的关键词。
- **allURLs**：`Record<string, SearchSnippet>`，所有已发现的URL及其内容。
- **visitedURLs**、**badURLs**：已访问/无效的URL列表。
- **evaluationMetrics**：`Record<string, RepeatEvaluationType[]>`，每个问题的评估指标。
- **finalAnswerPIP**：`string[]`，评审/改进建议，注入Prompt。

## 三、上下文的动态管理与更新

### 1. 初始化

- 在 `getResponse` 开始时，初始化所有上下文变量。
- 支持传入历史上下文，实现多轮对话/连续推理。

### 2. 每轮推理的上下文更新

- 每执行一步 action（如 answer、search、reflect、visit、coding），都会：
  - 记录 action 及其结果到 `allContext`
  - 更新 `allKnowledge`（如新知识、反思、事实等）
  - 更新 `gaps`（如新发现的子问题、已解决的gap移除）
  - 更新 `diaryContext`（推理过程日志）
  - 更新 `allURLs`、`visitedURLs`、`badURLs`（外部检索内容）
  - 更新 `evaluationMetrics`（评估反馈）
  - 更新 `finalAnswerPIP`（评审建议）

### 3. 上下文的持久化与可追溯性

- 每轮推理后，调用 `storeContext`，将 prompt、schema、allContext、allKnowledge、allQuestions、allKeywords、weightedURLs、msgWithKnowledge 等**持久化到本地文件**，便于调试和复盘。

```ts
await fs.writeFile('context.json', JSON.stringify(allContext, null, 2));
await fs.writeFile('knowledge.json', JSON.stringify(allKnowledge, null, 2));
await fs.writeFile('queries.json', JSON.stringify(allKeywords, null, 2));
await fs.writeFile('questions.json', JSON.stringify(allQuestions, null, 2));
```

### 4. 上下文在Prompt生成中的作用

- 每轮Prompt生成时，都会**动态读取最新的上下文变量**（如知识库、评审建议、gaps、action状态等），拼接成结构化Prompt，驱动LLM做出更优决策。

---

## 四、上下文管理的优势

1. **多步推理连贯性**：上下文贯穿全流程，保证每一步都能参考历史决策和知识。
2. **自我进化与反思**：评估反馈、反思内容、知识库等都在上下文中累积，驱动Agent持续自我修正。
3. **外部知识整合**：所有检索到的URL、内容、引用都纳入上下文，便于后续引用和聚合。
4. **资源可控**：TokenTracker等机制保证资源消耗可控，防止超预算。
5. **可追溯性与调试便利**：每轮上下文都可持久化，便于复盘、调优和问题定位。
6. **支持多轮对话/团队协作**：上下文可传递和复用，支持复杂多轮推理和团队分工。

---

## 五、迁移到你项目的建议

- **抽象出Context对象/类**，集中管理所有推理相关的状态信息。
- **每轮推理都用最新的Context生成Prompt、决策action、更新状态**。
- **持久化关键上下文变量**，便于调试和复盘。
- **上下文结构要支持扩展**，如后续增加新action、新知识类型等。
- **上下文与Prompt生成、action执行、评估反思等模块解耦**，便于维护和扩展。

---

## 六、总结

DeepResearch 的上下文管理机制，是其多步推理、动态决策、自我进化等能力的基石。  
**核心要点：全流程动态累积、结构化管理、可追溯、可扩展、与Prompt/决策/评估深度耦合。**
