在这个系统中，**agent 之间共享的核心内容主要包括：**

---

### 1. **上下文信息（Context）**
- 通过 `context_manager`（如 `ContextState`）统一管理。
- 包含：用户ID、会话ID、线程ID、全局上下文、各agent上下文、共享内存、对话历史、研究发现等。
- 作用：保证多智能体在同一会话下可以访问和补充同一份上下文，实现信息流转和多轮协作。

---

### 2. **用户画像与选择**
- 用户画像（如兴趣、标签、行为特征）由 ProfileAgent 等生成，写入上下文，供后续 agent 使用。
- 用户在每一步的选择（如洞察、画像、打点、内容等）通过 `nova3_selections` 结构化存储，所有 agent 可读取和补充。

---

### 3. **任务队列与执行状态**
- 所有 agent 共享同一个 `TaskQueue`，每个任务的状态、结果、输入输出都可被后续 agent 读取。
- 队列推进、任务完成、自动选择等状态由 supervisor 统一管理，agent 只需关注自身业务逻辑。

---

### 4. **历史结果与引用内容**
- 历史任务的结果（如分析、生成的内容）通过 `round_results_manager` 统一存储，支持 agent 之间引用和追溯。
- 支持 @引用机制，agent 可直接获取其他 agent 生成的内容片段。

---

### 5. **追踪链路（Tracing）**
- LangSmith 追踪ID（主链路ID、子链路ID）在 agent 之间传递，保证全流程可观测和调试。

---

### 6. **用户原始需求与补充输入**
- 用户的原始输入、补充说明等会写入上下文和队列，所有 agent 可用来定制自己的提示词和行为。

---

**总结：**  
agent 之间通过上下文管理器、任务队列、用户选择、历史结果、追踪链路等机制，实现了数据、状态、用户意图的共享和协作，保证了多智能体系统的高效协同和个性化输出。
